Rosetta is a paired-curve neural model for copper heap leach behavior, built to predict both a control recovery curve and a catalyzed recovery curve for the same ore under the same column conditions. Its core target is a biexponential recovery form, where a1 and a2 represent recoverable inventory split into a fast and slow pool, and b1 and b2 represent the fast and slow kinetic rates. The most important physical interpretation is that a1 + a2 is the ore’s maximum achievable recovery, a1 is the accessible fast inventory, a2 is the slower recoverable inventory or tail, b1 is the fast-branch kinetics, and b2 is the slow-tail kinetics. The model should always preserve the meaning of the branches: within any predicted parameter set, b1 must correspond to the faster branch and appear first, while a1 + a2 for the catalyzed case should be allowed to exceed the control case.

The model is not meant to be a generic black box. It is a metallurgically informed architecture in which ore mineralogy, chemistry, geometry, and catalyst exposure are routed through named latent interaction blocks. These latent blocks are intended to encode interpretable mechanisms such as fast leach inventory, oxide inventory, ferric synergy, passivation burden, depassivation, acid buffering, diffusion drag, surface refresh, and ore decay. A good Rosetta model should therefore behave like a constrained geometallurgical simulator: oxides, secondary sulfides, acid-soluble copper, cyanide-soluble copper, and bornite should generally support higher fast recovery and faster early kinetics; chalcopyrite-rich or primary sulfide-rich ores should increase passivation and slow-tail burden; carbonates, gangue silicates, coarse particle size, high density, tall columns, and adverse size-to-diameter ratios should suppress kinetics and late recovery; and pyrite or other acid-generating sulfides should mainly matter through ferric/redox assistance and catalyst responsiveness rather than by acting like easy copper inventory.

A critical design principle is that the latent interaction dictionary should not remain shared between control and catalyzed predictions necessarily (only in cases that apply). The ore physics do not change between scenarios; what changes is how catalyst modifies the response. For that reason, I think the preferred architecture is a tied-head design: the model predicts control parameters directly, then derives catalyzed parameters as a constrained modification of the control parameters using catalyst-relevant latents such as primary_catalyst_synergy, depassivation_strength, ferric_synergy, and inventory terms. One shared ore-physics prior, with catalyst represented as a structured delta layered on top of the control behavior. The catalyst should not create an entirely separate physics branch. It should mostly unlock additional slow recoverable inventory and relieve slow-tail resistance in passivated primary ores (chalcopyrite or residual as sometimes is called), with only secondary effects on fast kinetics.

The latent blocks should be organized by intended support: chem_raw should broadly support maximum recovery, fast_leach_inventory and surface_refresh should mostly support a1 and b1, while primary_passivation_drive, ore_decay_strength, acid_buffer_strength, and passivation_strength should mostly support a2 and b2 (this basically is because usually the catalyst is added after the fast leachable ore is leached, the oxides and part of the secondaries cyanide soluble). depassivation_strength and primary_catalyst_synergy should specifically support catalyst uplift in the slow tail. The model becomes more interpretable and more stable when each latent has a narrower job instead of several blocks redundantly trying to explain the same variance.

Rosetta’s prior dictionary can look richer than the actual executed behavior if the interaction code filters out too many direct terms. The model only behaves as intended if the latent interaction specifications are actually applied to the live interaction terms in the forward pass. It is also important not to over-constrain catalyst behavior. The only hard catalyst-side constraint that should remain is that catalyzed maximum recovery should be at least as large as control maximum recovery. The model should not force b1_cat > b1_ctrl, should not force b2_cat > b2_ctrl, but should force the catalyzed curve to be above control at every time point. It should only preserve internal branch ordering inside each predicted curve so that b1 always remains the fast branch.

In short, Rosetta should look like a paired, physics-guided neural model that predicts control leaching behavior and then expresses catalyst benefit as a mechanistically structured perturbation of that baseline. It should behave like a metallurgical reasoning engine: easy copper mineralogy should accelerate early recovery, passivating primary sulfides should load the slow tail, gangue and geometry should penalize kinetics, and catalyst should mainly unlock chalcopyrite-dominated, passivated recovery rather than acting like a universal fast-kinetics boost.

Also it is important to highlight that the model should understand and infer complex metallurgical effects, so all weights and specially LATENT_INTERACTION_SPECS should be consistent with the metallurgical knowledge of bioleaching systems to extract copper. The catalyst affects mainly the chalcopyrite (residual) which is usually passivated in these systems.
